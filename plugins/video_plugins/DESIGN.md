# 生成请求日志安全设计

## 适用范围

`common.py` 提供图片与视频生成共用的 HTTP 请求日志能力。协议插件仍负责构造和发送真实请求，
日志 helper 只生成供画布排障使用的脱敏快照，不参与鉴权、重试策略或业务结果解析。

## 数据流

1. 后端在发送生成请求前构造请求快照。
2. 请求完成或抛出网络异常后补充响应快照。
3. 快照通过任务 `request_details` 字段返回普通画布和智能画布。
4. 前端再次执行防御性脱敏，并将日志保存在对应画布数据中。

`request_details.transport` 固定为 `backend_http`，`attempts` 最多保留最近 8 次真实提交尝试。
CLI 型图片平台没有应用内可观察的上游 HTTP 请求，继续显示本地任务请求与结果摘要。

## 威胁模型

生成请求可能包含以下不应持久化的数据：

- API Key、Authorization、Cookie、Token、密码和其他凭据。
- URL 查询参数或片段中的签名。
- data URL、base64 图片、上传文件内容及其他大段二进制数据。
- 响应中的 Set-Cookie 或未筛选诊断头。

## 安全决策

- 敏感请求头统一替换为占位符，真实值仅用于从其他文本中二次替换。
- HTTP URL 只记录协议、主机与路径，不记录用户信息、查询参数和片段。
- 文件只记录字段名、文件名、媒体类型与大小，不记录文件内容。
- 响应头仅保留 Content-Type、Retry-After、请求 ID 和追踪 ID。
- 请求体与响应体限制字符串长度、数组数量和递归深度；常见 base64 字段及长内嵌
  `data` 字段只记录省略说明与原长度。
- 日志详情前端继续执行第二层脱敏，兼容旧日志和未来协议字段。

## 信任边界

上游响应、平台配置和生成参数均按不可信输入处理。后端脱敏结果进入 `CANVAS_TASKS` 后才允许
返回前端；前端不得接收原始 API Key 或原始二进制内容。用户提示词属于画布业务数据，会按现有
日志功能持久化，不视为凭据。

## 已知风险

- 未使用常见字段名、且内容不像 URL 或 base64 的业务秘密只能依赖字段名规则识别。
- Responses 的 SSE 流式尝试由流处理器消费；日志至少记录进入流式回退前的后台请求及后续
  非流式回退，当前不保存完整 SSE 事件流。
- 旧画布日志没有 `request_details`，只能继续显示本地请求摘要。

## 验证

- `tests/test_video_http_request_logging.py` 覆盖请求、响应、异常、重试和 multipart 脱敏。
- `tests/test_image_http_request_logging.py` 覆盖图片请求接入、成功/失败任务透传和内嵌数据省略。
- `tests/test_generation_log_detail.js` 覆盖前端日志详情的二次脱敏与 cURL 生成。
